# E2C2 Secure Access Configuration

## Overview

Performed access remediation and security hardening activities for the E2C2 Specialist Scorecard Application within Microsoft Entra ID.

The application was identified as having broader accessibility than intended. The objective was to restrict access to approved users while aligning permissions with least-privilege security principles.

## Responsibilities

- Investigated Enterprise Application configuration within Microsoft Entra ID
- Reviewed assigned users and groups
- Identified accounts requiring removal from application access
- Removed unauthorized access assignments
- Verified approved users retained required permissions
- Validated application access after remediation
- Coordinated privileged access activation through Microsoft PIM when elevated permissions were required

## Technologies Used

- Microsoft Entra ID
- Enterprise Applications
- Microsoft Privileged Identity Management (PIM)
- Azure RBAC
- Microsoft 365 Administration

## Skills Demonstrated

- Identity and Access Management (IAM)
- Application Access Control
- Security Remediation
- Principle of Least Privilege
- Access Reviews
- Enterprise Application Administration
- Privileged Access Management

## Outcome

Successfully restricted application access to approved users, removed unauthorized assignments, and aligned permissions with organizational security requirements.
